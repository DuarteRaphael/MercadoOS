# Contributing to MercadoOS

Obrigado por contribuir com o MercadoOS! Este documento descreve como colaborar de forma eficiente e segura.

## Como Contribuir

1. Fork o repositório e crie uma branch a partir de `main`.
2. Faça mudanças pequenas e focadas.
3. Crie um commit com mensagem clara e seguindo Conventional Commits.
4. Abra um pull request com descrição do que foi alterado e por que.

## Padrão de Commits

Use Conventional Commits:

- `feat:` para novas funcionalidades
- `fix:` para correções de bugs
- `docs:` para documentação
- `style:` para formatação e estilo
- `refactor:` para refatorações
- `test:` para testes
- `chore:` para tarefas de manutenção

## Normas de Revisão

- Mantenha os PRs pequenos e fáceis de revisar
- Documente mudanças de arquitetura ou design
- Inclua referências às issues quando aplicável
- Garanta que não há código de produção quebrado

## Reportando Problemas

Crie issues claras com:

- Descrição do problema
- Passos para reproduzir
- Comportamento esperado
- Comportamento atual

## Código de Conduta

Antes de contribuir, leia `CODE_OF_CONDUCT.md`.
